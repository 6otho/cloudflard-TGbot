# 🤖 Telegram 双向客服机器人 (Cloudflare Worker 完美版)

![License](https://img.shields.io/badge/license-MIT-blue.svg)
![Platform](https://img.shields.io/badge/platform-Cloudflare%20Workers-orange.svg)
![Database](https://img.shields.io/badge/database-Cloudflare%20D1-blue.svg)

这是一个基于 **Cloudflare Workers** 和 **D1 数据库** 构建的高级 Telegram 双向客服/传话筒机器人。它不仅可以实现用户与管理员的完美隔离沟通，还内置了 **Web 网页端可视化大盘**，并且针对 Telegram 官方的各种奇葩底层限制（如花体字断层、隐藏隐私无法点击等）做了**终极适配与修复**。

---

## ✨ 核心亮点

*   **☁️ 零成本部署**：完全托管于 Cloudflare Workers，无需服务器，白嫖首选。
*   **📂 自动话题分流**：利用 TG 群组的“论坛模式”，为每个用户自动创建专属沟通话题（Topic）。
*   **📋 不死汇总卡片**：自动生成「用户资料卡汇总」贴，集中管理所有访客，误删也能瞬间自动重建。
*   **🖥️ Web 控制台**：根目录直接访问带权限校验的网页管理面板，实时查看数据概况。
*   **🛡️ 人机验证防骚扰**：内置自定义验证问答（默认 1+1=3），屏蔽低级广告机。
*   **⚙️ 极致防断层排版**：完美兼容用户设置的花体字（如 `𝙹𝚘𝙺𝙴𝚛`）以及极端隐私保护设置，确保主页直达链接 100% 可用。
*   **🛠️ 快捷功能按钮**：资料卡内嵌快捷指令，一键支持 **置顶、禁言、静音、删档重置**。

## 🛠️ 准备工作 (准备好这 3 样东西)

在开始部署之前，你需要提前准备好以下信息：
1. **一个 Cloudflare 账号**（已开通 Workers & Pages 功能）。
2. **一个 Telegram Bot Token**：找 [@BotFather](https://t.me/BotFather) 创建一个新的机器人，并获取它的 `HTTP API Token`。
3. **一个作为“客服后台”的 TG 群组**：
   * 必须将该群组升级为 **超级群组 (Supergroup)**。
   * **⚠️ 必须在群设置中开启「Topics (论坛/话题) 模式」**。
   * 把你的机器人拉进这个群，并给予 **管理员权限**（至少包含：管理消息、管理话题权限）。
   * 获取这个群的 **Group ID**（一般是以 `-100` 开头的一长串数字，可通过向 [@RawDataBot](https://t.me/RawDataBot) 转发群消息获取）。

## 🚀 保姆级部署教程

### 第一步：创建 D1 数据库
1. 登录 Cloudflare 控制台，在左侧菜单找到 **Workers 和 Pages** -> **D1 SQL 数据库**。
2. 点击右上角的 **创建数据库**。
3. 数据库名称随便填（例如：`tg_bot_db`），点击创建。
4. **🎉 注意：你不需要手动建表！** 代码内置了自动建表功能（dbMigrate），部署后会自动搞定。

### 第二步：创建 Worker 服务
1. 在左侧菜单找到 **Workers 和 Pages** -> **概述**。
2. 点击 **创建** -> **创建 Worker**。
3. 为你的 Worker 起个名字（例如 `tg-bot-worker`），然后点击 **部署**。
4. 部署成功后，点击 **编辑代码**。
5. 将本项目仓库中的完整 `worker.js` 代码复制，覆盖掉网页编辑器里的所有代码，点击右上角的 **保存并部署**。

### 第三步：配置环境变量 (重中之重)
回到该 Worker 的详情设置页面，找到 **设置 (Settings)** -> **变量和机密 (Variables and Secrets)**，添加以下变量：

| 变量名 (Key) | 变量值示例 (Value) | 说明 |
| :--- | :--- | :--- |
| `BOT_TOKEN` | `12345:AAH...` | 机器人的 API Token |
| `ADMIN_GROUP_ID` | `-1001234567890` | 客服后台群组 ID（**必须带 `-100` 前缀**） |
| `ADMIN_IDS` | `123456789` | 主管理员的私人 TG ID（用于登录网页后台和免验证） |
| `WELCOME_MSG` | `欢迎！请先完成人机验证。` | (可选) 新用户发送 /start 时的欢迎语 |
| `VERIF_Q` | `问题：1+1=?` | (可选) 人机验证的问题 |
| `VERIF_A` | `3` | (可选) 人机验证的正确答案 |

### 第四步：绑定 D1 数据库
1. 依然在 Worker 的详情页面，找到 **设置 (Settings)** -> **绑定 (Bindings)**。
2. 点击 **添加绑定** -> 选择 **D1 数据库**。
3. 变量名称（Variable name）**必须**填入：`TG_BOT_DB` （不要乱改，必须大写）。
4. 数据库空间（Database namespace）选择你在【第一步】中创建的那个数据库。
5. 点击 **部署** 以保存绑定。

### 第五步：一键激活机器人 (绑定 Webhook)
1. 在浏览器中打开你刚刚部署好的 Worker 域名（例如：`https://tg-bot-worker.你的前缀.workers.dev`）。
2. 你会看到一个漂亮的 **传话筒管理中心** 登录界面。
3. 输入你在环境变量里设置的 `ADMIN_IDS` 进行登录。
4. 进入大盘后，在中间蓝色的系统状态控制区域，点击 **「⚡ 一键激活机器人」**。
5. 弹出提示“✅ 激活成功！”后，你的机器人就已经正式上线了！

## 🎮 使用指南

### 🧑‍💻 用户视角
1. 搜索机器人并发送 `/start`。
2. 机器人会弹出设置好的人机验证问题。
3. 用户回复正确的答案后，验证通过，此后发送的任何文字、图片、视频都会直接转发到后台客服群。

### 👮‍♂️ 管理员/客服视角
1. 用户发来消息时，机器人在后台群组自动创建一个以该用户命名的 **独立话题 (Topic)**。
2. 同时会在「📋 用户资料卡汇总」这个话题里推一张该用户的详细资料卡（支持置顶、禁言、重置等功能）。
3. 管理员只需要在**对应的独立话题内直接回复**，机器人就会自动把消息传达给该用户。

## 🗑️ 关于「删档重置」功能
在用户的资料卡下方有一个 **「🗑️ 重置(删档)」** 按钮。
这是一个非常实用的测试与管理功能：
* 点击后，系统会**彻底抹除**该用户在数据库中的所有记录，并关闭当前的话题框。
* 该用户下一次给机器人发消息时，将被系统当作全新的陌生人，**强制要求重新进行人机验证**。
* 非常适合你自己用大号反复调试机器人流程，或者清理已经结束服务的过期用户。

## ❓ 常见问题排查 (FAQ)

**Q1：网页后台点击「一键激活」提示失败？**
* 检查你的 `BOT_TOKEN` 是否填错，且没有多余的空格。
* 检查 Worker 域名是否被墙（国内访问 `*.workers.dev` 可能受限），建议在 CF 后台给 Worker 绑定一个自定义域名 (Custom Domain)。

**Q2：用户发消息后，机器人提示：“抱歉，暂时无法连接客服（创建话题失败）”？**
* **绝对原因 1**：你的后台群没有开启「Topics (论坛)」功能。请去群设置里打开它。
* **绝对原因 2**：你的机器人没有在这个群里被设置为“管理员”，或者没有赋予“管理话题(Manage Topics)”的权限。
* **绝对原因 3**：你的 `ADMIN_GROUP_ID` 填错了，超级群组记得务必带上 `-100`。

**Q3：点击资料卡上的「🔗 查主页」毫无反应？**
* 这是正常的 Telegram 官方隐私限制机制。说明该用户**没有设置用户名 (@xxx)**，且他在隐私设置中**禁止了别人通过转发链接查看他的主页**。此时你无法主动看他主页，只能通过机器人直接回复他。
